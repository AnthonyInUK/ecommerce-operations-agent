# Multi-turn Follow-up Demo Chain

这份文档展示的不是“系统会不会聊天”，而是：

**这个 Agent 已经开始能沿着同一个业务问题连续往下问，而不是每句都要求用户重新说完整。**

## 1. 演示目标

用一组最接近真实业务沟通的话，证明系统已经开始具备连续分析能力：

1. `昨天 GMV 多少？`
2. `那华东呢？`
3. `那女装呢？`
4. `那退款率呢？`

这组问题对应的真实业务含义是：

- 先看大盘
- 再缩到区域
- 再缩到品类
- 最后切到成交质量指标

也就是从“看总盘”一路走到“看某个业务切片的质量”。

## 2. 这条链在业务上证明了什么

它证明的不是“模型会记忆所有内容”，而是：

1. 系统会继承上一轮的日期
2. 系统会继承上一轮的区域
3. 系统会继承上一轮的品类
4. 系统能在 follow-up 里切换指标

换句话说，后面三句都不是完整句子，但系统仍然能知道你在继续问同一件事。

## 3. 当前会话链是怎么理解这四句的

### Step 1. `昨天 GMV 多少？`

系统识别：

- 指标：GMV
- 日期：昨天
- scope：大盘

### Step 2. `那华东呢？`

系统继承上一轮：

- 指标仍然是 GMV
- 日期仍然是昨天

只替换：

- 区域 = 华东

### Step 3. `那女装呢？`

系统继承上一轮：

- 指标仍然是 GMV
- 日期仍然是昨天
- 区域仍然是华东

只新增：

- 品类 = 女装

### Step 4. `那退款率呢？`

系统继承上一轮：

- 日期仍然是昨天
- 区域仍然是华东
- 品类仍然是女装

只切换：

- 指标 = 退款率

## 4. 当前这条链背后说明了什么设计

它依赖的是三层能力：

1. **会话状态**
   - 记住上一轮的日期、区域、品类、指标和意图

2. **指标词典**
   - 识别“退款率”是一次指标切换

3. **语义模型**
   - 识别“那华东呢 / 那女装呢 / 那退款率呢”这些 follow-up 的业务含义

## 5. 这条链当前还存在哪些边界

### 1. 只记最小上下文，不记完整世界

当前会话状态更多是：

- 上一轮意图
- 上一轮指标
- 上一轮日期
- 上一轮区域
- 上一轮品类

所以它更像“够用的最小记忆”，不是完整长期记忆。

### 2. 如果用户突然跨域，很可能继承错上下文

比如刚问完 GMV，再突然说：

- `那 ROI 呢？`

如果没有足够明确的新上下文，系统可能会继续沿着交易分析上下文去理解。

### 3. 澄清问题之后还没有完整二次接续

比如：

- `活跃用户怎么样？`
- 系统先问：`是 DAU 还是活跃买家？`

如果后续回答特别短，目前这条最小链还不是完整的多轮 slot-filling 系统。

### 4. 历史数据覆盖有限

像：

- `去年同期呢？`

语言理解入口已经有了，但 demo 数据不一定有足够完整的同比历史。

## 6. 演示时怎么讲

### 开场一句话

> 这条演示不是证明系统会聊天，而是证明它已经开始能沿着同一个业务问题连续下钻。

### 口播重点

你要强调的不是：

- “它记住了所有内容”

而是：

- “它已经能记住最值钱的那几个业务参数：日期、区域、品类、指标”
- “这样运营就不用每次都重新说完整句子”

## 7. 这条链证明了什么

它证明这个项目已经从：

- 单轮问数

往前走到：

- 最小连续分析会话

这是后面继续做：

- 更完整的 semantic model
- 更稳定的澄清链
- 更复杂的 root cause 多轮追问

的基础。
