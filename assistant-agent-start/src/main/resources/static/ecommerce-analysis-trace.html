经营异常分析工作流

从“GMV 跌了”到“该谁处理、先查什么”

这个页面演示的是一条最小业务闭环：系统先发现经营异常，再自动跑 root cause， 最后把原因、证据、责任角色和通知草稿整理出来。技术链路仍然保留，但默认折叠在开发调试信息里。

当前默认演示问题

推荐先点“一键跑完整闭环”。它会模拟每天早上的自动巡检，不是等用户提问，而是主动发现异常并接上归因链。

业务主线 异常 -> 归因 -> 分发
最终交付 通知草稿
1
发现异常 GMV 巡检先判断华东是否真的明显下跌。
2
拆解原因 按区域、订单、用户、品类、漏斗、退款逐层排查。
3
分发责任 把不同信号分给平台、增长、类目、售后等角色。
4
形成交付 生成可复制到飞书/群聊/日报的通知草稿。
